Mobile CSS Fix Test

This page helps verify that the Slide 3 text box issue has been fixed.

What was fixed:

  • Removed generic .content-wrapper styles that affected all slides
  • Made CSS rules specific to each slide (#page1, #page2, etc.)
  • Added !important flags to prevent style bleeding
  • Ensured Slide 3 text box only appears on Slide 3

Test Instructions:

  1. Open the main site in mobile view (or on a mobile device)
  2. Navigate through all slides using scroll or touch
  3. Verify that the text box only appears on Slide 3
  4. Check that Slide 1 (hero) has no background box
  5. Confirm other slides display their content correctly

Key Changes Made:

/* Before - Generic rule affecting all slides */
.content-wrapper {
    width: calc(100% - 50px);
    max-width: 360px;
    /* ... */
}

/* After - Specific to certain slides only */
#page2 .content-wrapper,
#page4 .content-wrapper,
#page6 .content-wrapper {
    width: calc(100% - 50px);
    max-width: 360px;
    /* ... */
}

/* Slide 3 specific rules */
#page3 .content-wrapper {
    /* Unique styles for Slide 3 */
    background: rgba(0, 0, 0, 0.5) !important;
    /* ... */
}

/* Slide 1 prevention */
#page1 .content-wrapper {
    background: transparent !important;
    /* ... */
}
